Complete Usage Guide (All Major Features)

Initial Setup and Company Profile
  • Open Settings and fill company identity, GSTIN, address, logo, invoice/quotation numbering, and declaration terms.
  • Set backup folder outside system drive and enable auto cloud sync.
  • Run Test Backup + Cloud Sync before finishing setup so recovery protection is verified.
Sales, Quotations, and Invoice Workflow
  • Create quotation or invoice from Sales workspace, then add products/services and quantity units.
  • Use shortcut keys for faster operations: new invoice, quotation mode, and save actions.
  • Convert finalized quotations into invoices and track payment updates from history views.
Inventory and Purchase Controls
  • Add products with stock and purchase linkage to keep inventory movements auditable.
  • Use manual stock edit and stock reconciliation tools when mismatches are detected.
  • Export mismatch reports for verification and physical stock checks.
Cloud Backup, Restore, and Account Recovery
  • Configure Local Folder or Google Drive provider for automated backup copy operations.
  • Use Verify Cloud Backup to validate ZIP integrity, database quick check, and table presence.
  • Link a recovery email account, request OTP, verify OTP, and run restore-from-email on reinstall.
OTP Security and Delivery Providers
  • Choose OTP delivery provider: SMTP, SendGrid, or Mailgun.
  • OTP send uses cooldown protection and verify attempts are lock-protected after repeated failures.
  • Provider credentials are stored with encryption and can be managed from setup/settings panels.
RBAC, Audit Trail, and Compliance
  • Use role-permission matrix to grant granular rights (create users, reset password, settings updates, etc.).
  • Export/import RBAC policy for portability across environments.
  • Generate immutable audit packages with hash-chain verification and signature checks.
Diagnostics, Health Monitoring, and Operations
  • Run health monitor to inspect sync failures, retry queues, and system job status.
  • Use diagnostics center for root-cause hints when cloud sync or restore actions fail.
  • Execute command palette actions for quick operations and administrative workflows.
Update Process for Admins
  • Build installer and publish to this site downloads folder.
  • Update latest manifest with version, publish date, file, and release notes.
  • Share this page URL with users; they always get the newest installer from one place.

How This Website Works

  • The page reads update-site/releases/latest.json to discover the latest installer.
  • The installer EXE is served from update-site/downloads.
  • Release notes are loaded from the manifest and shown in the Release Notes section above.
  • The latest user manual is published as a downloadable HTML guide for the same release.
  • Comments are stored per browser in static hosting mode, which keeps the page Vercel-friendly.
  • You can replace the file on every release and keep this page URL unchanged.
  • The same manifest can also be reused by in-app update checkers.
